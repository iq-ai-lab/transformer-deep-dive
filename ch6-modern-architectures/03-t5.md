## 🔍 왜 이 설계가 Transformer 의 핵심인가

T5 는 **NLP task 통일의 milestone**:

1. **Text-to-text framework** — translation, summarization, QA, classification 모두 single seq2seq
2. **Span corruption** — BERT 의 token-level mask 보다 효율적
3. **Encoder-Decoder 의 모범** — cross-attention 의 effective use
4. **C4 dataset** — modern training data curation 의 토대

이 문서는 T5 의 **objective, architecture, modern relevance** 를 다룹니다.

## 📖 직관적 이해

### Text-to-Text Framework

```
Translation:
  Input:  "translate English to French: The cat is black."
  Output: "Le chat est noir."

Summarization:
  Input:  "summarize: <long text>"
  Output: "<summary>"

Classification:
  Input:  "cola sentence: The cat sleeps."
  Output: "acceptable"

QA:
  Input:  "question: ... context: ..."
  Output: "<answer>"
```

모든 task 가 **string in, string out** — single model, single recipe.

### Span Corruption (T5's MLM)

```
Original: "The cat sat on the mat in the garden"

Corrupt (random span 선택):
Input:    "The cat <X> on the <Y> in <Z> garden"
Target:   "<X> sat <Y> mat <Z> the"

→ Span 단위 (multiple tokens) corruption + reconstruction
```

BERT 의 single-token mask 와 다름 — span 단위.

### Encoder-Decoder Architecture

```
Encoder: bidirectional, processes input
   ↓
   z (encoder output)
   ↓
Decoder: causal, generates output
  - Self-attention (causal)
  - Cross-attention to z
  - FFN
```

Translation 의 natural fit — encoder 가 source, decoder 가 target.

## ✏️ 엄밀한 정의

### 정의 3.1 — T5 Architecture

- **T5-base**: 12 encoder + 12 decoder layer, $d = 768$, 220M params
- **T5-large**: 24/24, $d = 1024$, 770M
- **T5-3B**: 24/24, $d = 1024, h_{ff} = 16384$, 3B
- **T5-11B**: $d = 1024, h_{ff} = 65536$, 11B

특징:
- Pre-LN (Vaswani 의 post-LN 아닌 modified)
- Relative bucketed bias (Ch3-04) — no absolute PE
- ReLU FFN (initial), GeGLU (later)
- SentencePiece tokenization (32K vocab)

### 정의 3.2 — Span Corruption Objective

Random span 들을 single sentinel token 으로 replace:
$$
\text{Input: "the cat <X> on the <Y> garden"}
$$
$$
\text{Target: "<X> sat <Y> mat in"}
$$

**Loss**:
$$
L = -\sum \log p(\text{target} | \text{input})
$$

(autoregressive on target side)

### 정의 3.3 — Sentinel Tokens

Special tokens for span boundaries: `<extra_id_0>`, `<extra_id_1>`, ..., `<extra_id_99>`. 각 corrupted span 별 unique sentinel.

### 정의 3.4 — C4 Corpus

Colossal Clean Crawled Corpus:
- Common Crawl 의 cleaned subset
- ~750GB text (T5)
- Filtering: heuristic + language detection + dedup
- Modern model 들의 baseline (LLaMA 등이 derivative 사용)

### 정의 3.5 — Task Prefixes

Each task 의 prefix:
- "translate English to German: ..."
- "summarize: ..."
- "cola sentence: ..."
- "stsb sentence1: ... sentence2: ..."

→ Single model 이 prefix 로 task identify.

## 🔬 정리와 증명

### 정리 3.1 — Span Corruption 이 Token Masking 보다 효율

BERT 의 MLM:
- Loss 는 15% masked positions 만
- 85% positions 는 self-prediction (trivial) — 학습 signal 없음

T5 의 Span Corruption:
- Encoder 가 input 전체 처리 (bidirectional)
- Decoder 가 모든 corrupted span 예측
- Effective loss density ↑

**Empirical**: T5 가 BERT 대비 same compute 에서 더 좋은 representation.

### 정리 3.2 — Encoder-Decoder vs Decoder-only Sample Efficiency

**Sample efficiency 비교** (Wang 2022):

작은 scale (~1B):
- Encoder-decoder 가 specific task 에서 우수
- Translation, summarization

큰 scale (~10B+):
- Decoder-only + ICL 이 동등하거나 우수
- Architecture inductive bias 의 효과 약화

→ **Scale 의존**: encoder-decoder 의 advantage 가 small scale 에서 명확.

### 정리 3.3 — Cross-Attention 의 효과

Decoder 의 매 layer 가 encoder output 에 cross-attend → 직접적 conditioning.

Decoder-only 의 implicit conditioning (concatenation) 보다 explicit.

**Trade-off**:
- Encoder-decoder: more parameters (cross-attention), explicit conditioning
- Decoder-only: simpler, scale 에서 같은 effect emergent

### 정리 3.4 — Span Length Distribution

T5 의 corruption: average span length 3 tokens, 15% of input total.

**Why these hyperparameters**:
- Too short (1 token): like BERT — limited
- Too long (10+ tokens): too challenging
- 3 tokens: phrase-level pattern 학습

### 정리 3.5 — Universal Task Representation

Text-to-text 의 universality:
- Classification: output text "yes" / "no"
- Regression: output text number "0.5"
- Multi-label: output text comma-separated

**한계**: structured output (JSON, code) 도 text 그러나 generation quality 의존.

### 정리 3.6 — Modern T5 Variants

- **mT5** (multilingual): 101 languages
- **byT5** (byte-level): no tokenization
- **Flan-T5**: instruction-tuned
- **UL2** (Tay 2022): mixture of denoising objectives

각 variant 가 specific weakness 보완.

## 💻 NumPy / PyTorch 구현 검증

### 실험 1 — Tiny T5 구현

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np

class TinyT5Block(nn.Module):
    """Encoder block — bidirectional"""
    def __init__(self, d, h, d_ff):
        super().__init__()
        self.attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1 = nn.LayerNorm(d); self.ln2 = nn.LayerNorm(d)
    
    def forward(self, x):
        x = x + self.attn(self.ln1(x), self.ln1(x), self.ln1(x))[0]
        x = x + self.ffn(self.ln2(x))
        return x

class TinyT5Decoder(nn.Module):
    """Decoder block — causal self-attn + cross-attn + FFN"""
    def __init__(self, d, h, d_ff):
        super().__init__()
        self.self_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d, d_ff), nn.ReLU(), nn.Linear(d_ff, d))
        self.ln1, self.ln2, self.ln3 = nn.LayerNorm(d), nn.LayerNorm(d), nn.LayerNorm(d)
    
    def forward(self, x, encoder_out):
        T = x.size(1)
        causal = torch.triu(torch.ones(T, T), diagonal=1).bool()
        x = x + self.self_attn(self.ln1(x), self.ln1(x), self.ln1(x), attn_mask=causal)[0]
        x = x + self.cross_attn(self.ln2(x), encoder_out, encoder_out)[0]
        x = x + self.ffn(self.ln3(x))
        return x

class TinyT5(nn.Module):
    def __init__(self, vocab_size=1000, d=128, h=4, L=2):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, d)
        self.encoder = nn.ModuleList([TinyT5Block(d, h, 4*d) for _ in range(L)])
        self.decoder = nn.ModuleList([TinyT5Decoder(d, h, 4*d) for _ in range(L)])
        self.lm_head = nn.Linear(d, vocab_size, bias=False)
    
    def encode(self, src_ids):
        x = self.embed(src_ids)
        for layer in self.encoder:
            x = layer(x)
        return x
    
    def decode(self, tgt_ids, encoder_out):
        x = self.embed(tgt_ids)
        for layer in self.decoder:
            x = layer(x, encoder_out)
        return self.lm_head(x)
    
    def forward(self, src_ids, tgt_ids):
        enc_out = self.encode(src_ids)
        return self.decode(tgt_ids, enc_out)

torch.manual_seed(0)
model = TinyT5(vocab_size=1000, d=128, L=2)
src = torch.randint(0, 1000, (2, 16))
tgt = torch.randint(0, 1000, (2, 12))
logits = model(src, tgt)
print(f'T5 output: {logits.shape}')   # (2, 12, 1000)

# Param 비교 with decoder-only
print(f'T5-style params: {sum(p.numel() for p in model.parameters())/1e6:.2f}M')
```

### 실험 2 — Span Corruption 구현

```python
def span_corruption(input_ids, vocab_size, mask_ratio=0.15, mean_span_length=3, sentinel_start=900):
    """T5-style span corruption"""
    T = input_ids.size(0)
    n_corrupted = int(T * mask_ratio)
    n_spans = max(1, n_corrupted // mean_span_length)
    
    # Random non-overlapping spans
    span_starts = sorted(np.random.choice(T - mean_span_length, n_spans, replace=False))
    spans = []
    for s in span_starts:
        spans.append((s, min(s + mean_span_length, T)))
    
    # Build corrupted input and target
    input_corrupted = []
    target = []
    last_end = 0
    sentinel_id = sentinel_start
    
    for start, end in spans:
        input_corrupted.extend(input_ids[last_end:start].tolist())
        input_corrupted.append(sentinel_id)
        target.append(sentinel_id)
        target.extend(input_ids[start:end].tolist())
        last_end = end
        sentinel_id += 1
    
    input_corrupted.extend(input_ids[last_end:].tolist())
    target.append(sentinel_id)   # final sentinel
    
    return torch.tensor(input_corrupted), torch.tensor(target)

# Test
torch.manual_seed(0)
np.random.seed(0)
input_ids = torch.randint(0, 800, (40,))
corrupted, target = span_corruption(input_ids, 1000)

print(f'Original:  {input_ids.tolist()[:20]}...')
print(f'Corrupted: {corrupted.tolist()[:20]}...')
print(f'Target:    {target.tolist()[:20]}...')
# Sentinel tokens (900+) 가 spans 표시
```

### 실험 4 — Cross-Attention Visualization

```python
# Encoder output 의 어떤 부분에 decoder 가 attend?
torch.manual_seed(0)
src = torch.randint(0, 1000, (1, 8))
tgt = torch.randint(0, 1000, (1, 5))

# Get cross-attention weights
class TinyT5Trace(TinyT5):
    def get_cross_attention(self, src_ids, tgt_ids):
        enc_out = self.encode(src_ids)
        x = self.embed(tgt_ids)
        T_tgt = x.size(1)
        causal = torch.triu(torch.ones(T_tgt, T_tgt), diagonal=1).bool()
        cross_attn_weights = []
        for layer in self.decoder:
            x = x + layer.self_attn(layer.ln1(x), layer.ln1(x), layer.ln1(x), attn_mask=causal)[0]
            x_ln = layer.ln2(x)
            attn_out, attn_w = layer.cross_attn(x_ln, enc_out, enc_out, need_weights=True)
            cross_attn_weights.append(attn_w)
            x = x + attn_out
            x = x + layer.ffn(layer.ln3(x))
        return cross_attn_weights

t5_trace = TinyT5Trace(1000, d=64, L=2)
weights = t5_trace.get_cross_attention(src, tgt)
print(f'Cross-attention weights per layer: {[w.shape for w in weights]}')
# (B, T_tgt, T_src) — alignment matrix
```

## 🔗 실전 활용

### 1. T5 Variants

- **T5-base/large/3B/11B**: original
- **mT5** (Xue 2021): multilingual, 101 languages
- **byT5** (Xue 2022): byte-level, no tokenizer
- **Flan-T5** (Chung 2022): instruction tuned
- **UL2** (Tay 2022): mixture-of-denoisers

### 2. BART 와의 차이

BART (Lewis 2020): 비슷한 encoder-decoder 그러나:
- BART: token-level masking (BERT-like)
- T5: span corruption
- BART: BPE tokenization (vs T5 의 SentencePiece)
- BART: multilingual variant 있음 (mBART)

둘 다 modern era 에 less prominent (decoder-only 가 dominant).

### 3. Translation, Summarization

Encoder-decoder 가 이론적으로 적합:
- 명확한 input/output 분리
- Cross-attention 의 explicit alignment

그러나 modern LLM (GPT-4, Claude) 가 prompt-based 로 동등 또는 우수.

### 4. Modern Era 의 Encoder-Decoder

- **Whisper** (OpenAI): audio → text encoder-decoder
- **BLIP-2**: image → text 의 Q-former (encoder-decoder hybrid)
- **mBART, NLLB**: translation 의 specific use case

→ Multimodal / specialized translation 에서 niche.

### 5. Why Decoder-only Won (Frontier)

1. **Scale efficiency**: simpler architecture, easier scaling
2. **ICL**: few-shot 의 emergent
3. **Single objective**: causal LM 으로 모두 통일
4. **Data efficiency**: large unlabeled corpus 활용

T5 의 spirit 은 LLM 의 instruction tuning 으로 absorb.

## 🤔 생각해볼 문제

**문제 1** (기초): T5-base (220M) 의 architecture 구성요소별 파라미터 수를 계산하라. ($L=12+12, d=768, d_{ff}=3072$)

<details>
<summary>해설</summary>

**Encoder layer** (per layer):
- Attention: $4 \times 768^2 = 2.36M$
- FFN: $2 \times 768 \times 3072 = 4.72M$
- Per layer: ~7.1M

**Decoder layer** (per layer):
- Self-attention: $4 \times 768^2 = 2.36M$
- Cross-attention: $4 \times 768^2 = 2.36M$ (Q from decoder, K/V from encoder)
- FFN: $4.72M$
- Per layer: ~9.4M

**Total**:
- 12 encoder × 7.1M = 85.2M
- 12 decoder × 9.4M = 112.8M
- Embedding (32K vocab × 768): 24.6M
- Final LN, etc: small
- **Total: ~222M ≈ 220M** ✓

**Comparison**:
- Encoder layer 가 decoder 보다 작음 (no cross-attn)
- Decoder 가 ~30% 더 많음
- Encoder-decoder 가 GPT-style (decoder-only same total) 보다 약 50% 더 큼
- Trade-off: explicit conditioning vs model size

→ T5-base 이 BERT-base 와 비슷한 파라미터 (110M vs 220M, 그러나 effective compute 는 비슷). $\square$

</details>

**문제 2** (심화): T5 의 span corruption 이 BERT 의 token masking 보다 efficient 한 이유 — sample efficiency 와 effective loss density 의 정량 분석.

<details>
<summary>해설</summary>

**BERT MLM**:
- Input length $T$
- Masked tokens: 15% × $T$
- Loss only on masked positions

Per-batch effective signal: $0.15 \times T$ tokens

**T5 Span Corruption**:
- Encoder input: $T$ tokens (with sentinels) — full bidirectional context
- Decoder output: spans을 reconstruct — average span 3 tokens, 15% total
- Loss on **all decoder tokens** (sentinels + span tokens)

Per-batch effective signal: $0.15 \times T$ tokens (similar)

**Then why is T5 better?**

1. **Decoder 의 autoregressive learning**:
   - 각 corrupted span 의 reconstruction 이 sequence task
   - "<X>" 다음 무엇? → autoregressive 다음 token 예측
   - Generation 능력 학습 — BERT 는 generation 못함

2. **Span-level context**:
   - Phrase-level pattern 학습 (multi-word collocation)
   - Token-level mask 보다 context-rich

3. **Encoder-decoder communication**:
   - Cross-attention 학습 — translation, summarization에 useful
   - BERT 의 single-stream 보다 더 rich representation

4. **Sentinel 을 통한 explicit task**:
   - 각 span 의 reconstruction 이 explicit objective
   - BERT 의 implicit "fill in the blank" 보다 명확

**Empirical**:

Raffel 2020 의 ablation:
- T5 의 span corruption vs BERT 의 token masking
- Same compute 에서 T5 가 GLUE, SuperGLUE 에서 우수
- Translation 에서는 명확한 차이 (BERT 못함)

**Modern view**:

- Decoder-only LLM (GPT-3+) 은 다른 paradigm — span corruption 없음, 단순 next-token prediction
- 그러나 next-token prediction + scale 가 span corruption 의 효과 absorb
- T5 의 logical conclusion 이 modern LLM 의 instruction tuning

**Sample Efficiency**:

T5 의 span corruption 이 small/medium scale (1B 이하) 에서 우수. Large scale (10B+) 에서는:
- Decoder-only + ICL 이 동등 또는 우수
- Architecture inductive bias 의 advantage 약화

따라서 T5 의 우수성은 **specific scale + objective design** 의 sweet spot. Frontier 에서는 simpler decoder-only 가 dominant. $\square$

</details>

**문제 3** (논문 비평): Modern era (2024+) 에서 encoder-decoder (T5, BART) 가 frontier 에서 사라지고 decoder-only 가 dominant 한 이유. 그러나 Whisper, BLIP-2 같은 specific application 에서 여전히 사용되는 이유는?

<details>
<summary>해설</summary>

**Decoder-only 의 frontier dominance 이유**:

1. **Architecture simplicity**:
   - Single block type — 균일한 scaling
   - Encoder-decoder 의 두 part 의 별도 tuning 불필요
   - Implementation, debugging 더 simple

2. **Compute efficiency at scale**:
   - Wang 2022: same compute 에서 decoder-only 가 거의 동등
   - Encoder 의 추가 50% 파라미터가 frontier scale 에서 marginal benefit

3. **In-Context Learning 의 emergence**:
   - GPT-3 의 ICL 이 implicit conditioning 의 emergent
   - Encoder-decoder 의 explicit cross-attention 보다 더 flexible
   - Few-shot, instruction following 자연스러움

4. **Single objective**:
   - Causal LM 이 모든 task 의 framework
   - T5 의 "task prefix" 와 비슷하지만 more flexible
   - RLHF 같은 alignment 도 decoder-only 가 simpler

5. **Production convenience**:
   - KV cache, generation pipeline 통일
   - vLLM 같은 framework 가 decoder-only 우선

**Encoder-Decoder 의 specific application 가치**:

1. **Whisper (Speech Recognition)**:
   - Audio → text 의 명확한 input/output
   - Encoder 가 audio mel-spectrogram 처리
   - Decoder 가 text generation
   - Cross-attention 이 audio frame ↔ text alignment

2. **BLIP-2 (Vision-Language)**:
   - Q-former: vision encoder + learnable query
   - Cross-attention 이 image features 를 text decoder 에 전달
   - Frozen LLM + adaptable Q-former

3. **Translation Specific**:
   - NLLB-200, M2M-100 등 specific translation 모델
   - Encoder-decoder 의 alignment advantage
   - Decoder-only 도 가능하지만 specific task 에는 encoder-decoder 가 efficient

4. **Structured Generation**:
   - Code, JSON 등 structured output
   - Encoder 의 input 분석 + decoder 의 generation 분리
   - 일부 specialized model

**Why niche, not frontier**:

- Frontier model 은 generality 추구 (GPT-4, Claude, Gemini)
- Specific task 의 specialization 은 fine-tuning + prompt 로 처리 가능
- Multimodal (BLIP-2 같은) 은 still encoder-decoder 가 자연스러움

**Modern Hybrid**:

- LLaMA-Adapter 등: Frozen LLM + adapter (encoder-decoder spirit)
- Multimodal: vision encoder + LLM decoder
- Tool use: external retriever (encoder) + LLM (decoder)

**미래**:

- Frontier: still decoder-only + adapter for specific
- Multimodal frontier: complex hybrid
- Specialized application (translation, ASR): encoder-decoder 유지 가능

**근본 통찰**:

Encoder-decoder 의 advantage (explicit conditioning, alignment) 가 **scale + RLHF + ICL** 로 absorbed. 그러나 multimodal, specialized application 에서는 still useful.

T5 의 spirit (text-to-text framework) 는 **LLM instruction tuning** 으로 진화 — model architecture 가 아니라 prompt format 로. $\square$
